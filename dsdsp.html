検出された問題点

【問題 1】LST001_大リスト8 でフォントが混在
タイトル部（LB_タイトル1〜3、LB_タイトル_減点、LB_タイトル_Total）は Segoe UI Black、 明細部（LB_結果1〜8 の各列）は Segoe UI Semibold が使われており、1ファイル内でフォントが異なります。 他の LST004/LST005/LST006 はすべて Segoe UI Semibold で統一されています。
【問題 2】TIT001 で FontFamily と FontWeight の二重指定
Segoe UI Semibold は既にセミボールドのフォントフェイスであるにもかかわらず、 LB_Title1・LB_Title2 に FontWeight="Bold" が追加されています。 WPF は存在しない Bold ウェイトを合成（Synthetic Bold）で補完するため、見た目が過剰になる可能性があります。
【問題 3】LB_演技順 / LB_種目順 系ラベルの FontFamily 未指定
TIT002〜TIT007 内の LB_演技順・LB_種目順 は FontWeight="Bold" のみが指定され、 FontFamily が設定されていません。周囲のラベルが Segoe UI Semibold を明示指定している中で これらだけシステムデフォルト（Segoe UI Regular）が適用されています。
【問題 4】COM001〜003 の FontFamily 未指定
共通パーツ（左上・右上）は FontFamily が指定されておらず、システムデフォルトが適用されています。 他のパーツと視覚的な一貫性を持たせるには明示指定が望ましい状態です。

推奨対応

優先度対応内容対象ファイル
高 タイトル部の Segoe UI Black を Segoe UI Semibold に統一する（または意図的な使い分けであれば LST004 等にも適用する） LST001_大リスト8
中 Segoe UI Semibold + FontWeight="Bold" の二重指定を解消する（FontWeight を削除） TIT001_中央タイトル大
中 LB_演技順・LB_種目順 に FontFamily="Segoe UI Semibold" を追加し、FontWeight="Bold" を削除する TIT002〜TIT007（6ファイル）
低 COM001〜003 に FontFamily="Segoe UI Semibold" を追加して統一する COM001, COM002, COM003
参考 App.xaml の Application.Resources に Style TargetType="Label" でデフォルト FontFamily を定義すれば個別指定が不要になり保守性が向上する App.xaml
